how data reaches Claude & how Claude injects it into responses
static reference data
Prakriti percentages
Injected into system prompt via context block. Claude reads
"vata: 65%, pitta: 25%, kapha: 10%" and
references these directly in prose.
Frontend gets Prakriti independently via
GET /api/assessment/prakriti
Does NOT rely on Claude to pass it back.
Claude's prose references it; the app
renders the DB record directly.
dynamic live data
Vikriti / check-in trends / wearable summary
Assembled fresh per request from DB
and injected into dynamic context block.
Claude references current numbers in prose:
"Your HRV has been declining 3 days..."
Frontend renders live state from
GET /api/insights/health
Independently. Claude's summary
adds interpretation, the app adds
the actual charts and gauges.
on-demand knowledge lookup
Herb properties / food Ayurvedic data
Claude calls lookup_herb() or lookup_food() tool.
Tool checks herbs/foods cache table.
If not cached → Claude generates from
training knowledge → result cached.
Claude embeds data in prose response.
No separate frontend fetch.
Herb/food data lives only in
Claude's prose response + cache.
Cache = herbs / foods tables
ai_generated = true until reviewed.
