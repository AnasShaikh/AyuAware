2. Interaction Flows v2

     
Sign Up    
User Request UI Response Backend Operations
User clicks Sign Up Sign up options screen opens (Google/Apple/Email) Route to signup screen
User selects provider Provider-specific auth form opens Initiate provider auth flow and validate callback
User completes signup User lands in onboarding start Create user, issue tokens, set onboarding_complete=false
     
Login    
User clicks Login Login options screen opens Route to login screen
User submits credentials/provider auth If onboarding incomplete -> Onboarding; else -> Today Validate session and route by onboarding_complete flag
     
Onboarding    
User submits onboarding details Progresses to next onboarding step or missing-data prompt Score constitution, save profile/state, enforce required fields
User connects wearable (optional in base PRD) Connection status shown Register wearable source and enqueue initial sync
User completes onboarding chat Protocol preview shown Persist onboarding conversation and generate first protocol draft
User accepts first protocol Redirect to Today screen Activate protocol and mark onboarding_complete=true
     
Today    
User opens Today Morning summary + status + today actions rendered Fetch latest summary, current state, active protocol, today tracking state
User marks an action complete Action tile updates immediately Upsert protocol_tracking and recalculate adherence
User starts daily check-in Check-in modal and insight response shown Save check-in, update Vikriti, return quick insight
     
My Twin    
User opens My Twin Twin overview loads Fetch constitution/current state/preferences and recent context
User navigates to Intelligence Evolution / Patterns / Predictions Corresponding card/slide renders Generate and cache card payloads from user context
User customizes Twin style Updated Twin style reflected in UI Update twin preferences
     
Health    
User opens Health screen Health dashboard cards render Fetch insights/health and state summaries
User opens metric detail (sleep, HRV, pillars) Detail views and trends render Fetch insights/wearables and trend windows
     
Exposure    
User opens Exposure Environmental impact card renders Resolve location/date + profile context, generate exposure guidance
User browses Light/Lunar/Astrology/Focus cards Carousel cards render Load or generate card-specific interpretation payload
     
Profile    
User opens Profile Profile sections render Fetch users/me + health profile + preferences + subscription summary
User updates goals/preferences Saved confirmation state shown Validate and persist profile/preference updates
User configures notifications Notification setting toggles persist Save device token/settings and update notification prefs
     
Nutrition    
User opens Nutrition Nutrition overview and guidance render Fetch nutrition insights + meal log summary + constitution context
User logs meal (manual/photo) Meal log entry appears Save meal log, trigger optional scan analysis, refresh nutrition summary
     
Supplements    
User opens Supplements Schedule/effectiveness/safety cards render Fetch supplement recommendations and interaction checks
User requests supplement change Updated recommendation shown Run safety check and save protocol modification if approved
     
Lifestyle    
User opens Lifestyle Routine and analytics cards render Fetch lifestyle logs + check-in trends + wearable context
User starts a guided session/practice log Session UI and completion state shown Store session/practice log and update adherence signals
     
Projects    
User opens Projects Project list and progress render Fetch projects and linked protocol progress
User creates a new project New project detail page opens Create project record and initialize milestones
User requests recommendations in project Recommendation block renders Generate recommendations from project goal + health context
